## 🔍 **DACA/AdalQarau** — Охотник за коррупционерами

Мой главный pet-проект. Создаю систему, которая помогает находить коррупционные схемы в госзакупках Казахстана.

**💡 Как это работает**: Система строит граф связей между компаниями и находит подозрительные паттерны — одни и те же участники, фиктивные конкуренты, завышенные цены.

#### **✨ Ключевые возможности**

- **🕸️ Граф-анализ связей** — выявление скрытых связей между участниками тендеров
- **⚠️ Риск-индикаторы** — автоматическое выявление подозрительных паттернов в закупках
- **📊 Аналитические дашборды** — визуализация тенденций и аномалий в данных
- **🔍 Полнотекстовый поиск** — быстрый поиск по документам и участникам
- **📋 Excel-экспорт** — выгрузка аналитических отчетов для работы следователей
- **📈 Статистика по регионам** — региональная аналитика закупочной активности

#### **🏗️ Архитектура и технологии**

**Backend:**

- **Микросервисная архитектура** на Go с gRPC взаимодействием
- **PostgreSQL** для основного хранилища + полнотекстовый поиск
- **MinIO** для хранения документов и файлов тендеров
- **GraphQL API** для гибкого взаимодействия с фронтендом
- **ProtoBuf** + Buf для управления схемами данных

**DevOps:**

- **Docker Compose** для локальной разработки
- **Makefile** для автоматизации развертывания
- **GitHub Actions** для CI/CD пайплайнов

#### **💡 Технические решения**

- **Граф-база данных** на PostgreSQL с recursive CTE для анализа связей
- **Event Sourcing** для отслеживания изменений в тендерных данных
- **Кеширование** частых запросов для улучшения производительности
- **Batch processing** для обработки больших объемов данных о закупках

#### **📈 Impact & Результаты**

- ✅ **Используется правоохранительными органами РК** для расследований
- ✅ **Выявлено 50+ подозрительных тендеров** за первые месяцы работы
- ✅ **Автоматизированы процессы** анализа, которые раньше занимали недели
- ✅ **Повышена эффективность** работы следственных органов

## 🏢 Как я оптимизирую системы в Ozon

### **"Матрица"** — когда запросы тормозят на 30 секунд

Помните фильм "Матрица"? Вот и у нас была своя "Матрица" — HR-платформа, которая подвисала на полминуты при каждом клике.

**🎯 Задача**: 1000+ HR-специалистов ждали по 30 секунд каждый запрос. Нужно было что-то делать!

**🔧 Что я сделал**:

- **SQL-запросы**: Разобрал медленные запросы и переписал с нуля → **ускорил в 8 раз**
- **ElasticSearch**: Поправил индексы и агрегации → **поиск работает в 2.5 раза быстрее**  
- **Кеширование**: Добавил Redis для частых запросов → **нагрузка на БД упала**

**🎉 Результат**: Теперь то, что занимало 30 секунд, работает за 4 секунды!

### **Staff Portal** — разбираю монолит на микросервисы

**🎯 Ситуация**: Огромный монолит "портал сотрудников" начал трещать под нагрузкой.

**🔧 Решение**: Аккуратно выделил микросервис управления сотрудниками и создал staff-lib — библиотеку, которую теперь используют 10+ сервисов команды.

**🎉 Результат**: Нагрузка упала на 25%, а новые фичи разрабатывать стало гораздо проще!
